### Why contribute to Git?
Git is the tool every developer uses daily. Contributing to it is a way to give back to the entire industry. The codebase is mature, strict, and follows a unique mailing-list-based contribution workflow (though GitHub mirrors exist).

### Contribution Tips
- **Mailing List**: Git uses a traditional email-based patch workflow. Learn how to use `git format-patch` and `git send-email`.
- **Portability**: Git must run on everything from ancient servers to modern macOS. Avoid non-standard C features.
- **Documentation**: Improving the `man` pages is a great way to start.
- **Refactoring**: There are ongoing efforts to move internal logic from shell scripts to C built-ins.
