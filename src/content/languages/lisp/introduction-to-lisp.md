## What is Lisp?

**Lisp** (List Processing) is the second-oldest high-level programming language still in widespread use (after Fortran). It pioneered many features we take for granted today, including garbage collection, dynamic typing, and the "Read-Eval-Print Loop" (REPL).

```lisp
(print "Hello, Nocturnal Codex!")
```

## Why Lisp?

1.  **Code as Data (Homoiconicity)**: Lisp code is written in Lisp's own data structures (Lists). This allows the language to manipulate its own code with ease.
2.  **The Macro System**: Lisp macros are not just text replacement; they are real Lisp functions that transform the code before it is compiled.
3.  **Dynamic Development**: You can modify your running program while it's active.
4.  **Symbolic Processing**: Designed from the start to handle symbols and logic, making it the original language of Artificial Intelligence.

## Dialects

-   **Common Lisp**: The large, "Industrial Strength" standardized version.
-   **Scheme**: A minimalist, elegant dialect often used in academia.
-   **Clojure**: A modern Lisp built on the JVM (Java Virtual Machine).

## The REPL

Wait! The **REPL** is the heart of the Lisp experience. You don't "Write-Compile-Run"; you interact with the running image of your program, incrementally building and testing functions.
